20120511¶

Schuldnerberatung¶

Continued with docs/tickets/63.

  • Number formatting. lino.utils.appy_pod.Renderer.insert_table() (the central method used for printing tables) now formats numbers more or less correctly. Is respects lino.Lino.decimal_separator and the new setting lino.Lino.decimal_group_separator. Most of the work is done by lino.ui.extjs2.ext_store.StoreField.value2html() and lino.ui.extjs2.ext_store.StoreField.format_value(), and the new method lino.ui.extjs2.ext_store.StoreField.format_sum().

    Note: Zero values are currently always hidden (printed as empty cells, not “0” or “0,00”) It is not yet possible to configure this behaviour.
