Wednesday, June 11, 2014¶

Excerpts also for Così¶

DONE:

  • Added lino.modlib.excerpts to Lino Così. ml.sales.Invoice is now lino.modlib.excerpts.Certifiable.
  • Started to document ml.sales.
  • Renamed field “invoicing_address” to “invoice_recipient”
  • The new lino.modlib.excerpts.fixtures.std fixture automatically creates a certifying lino.modlib.excerpts.ExcerptType instance for every model which inherits from lino.modlib.excerpts.Certifiable.
  • Cool: the new method lino.core.site.Site.str2kw() used for the above fixture is also the beginning of a solution for docs/tickets/108.

So ml.sales.Invoice is no longer TypedPrintable but lino.modlib.excerpts.Certifiable. This currently means that every app which uses ml.sales must also install lino.modlib.excerpts. Hm... Can we leave this as this? Do we need automatic management of dependencies (i.e. lino.core.plugin.Plugin.requires)?

  • Another example why we need app dependencies is sepa and iban.
