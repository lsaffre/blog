Explanation: the dpy fixtures are of course executed before any the kernel setup has been done. That’s why there was no config object jetzt in the lino.Lino instance. And the site’s lino.models.SiteConfig.default_build_method has not yet been restored.

Solution: new configuration parameter lino.Lino.site_config_defaults. And instead of setting a config instance variable during kernel setup, we now have a property site_config which lazily calls get_site_config.
