20131127 (Wednesday, 27 November 2013)¶

Upgrade to testlino¶

Upgraded testlino in Eupen to the newest development version. This took more time than usual because most projects are now on github, so I had to adapt the “pull” procedure.

Tested the data migration. Updated lino_welfare.migrate.migrate_from_1_1_10() to remove field help_text in accounts.Group and accounts.Account.

A bug visible in the demo¶

The example demo application built by The Lino Polls tutorial also demonstrated a bug: answering the confirmation “X has already N votes! Do you still want to vote for it?” with “No” caused an Internal Server Error. The bug itself was in lino.core.requests.BaseRequest.confirm().

Upgraded :ref`lf` to the newest development version.

This revealed two more stale SITE instantiations in lino.projects.presto.settings and lino.projects.belref.settings
