Found a subtle (internal) bug:

When a ChoiceListField is empty¶

The welfare.pcsw.Clients table showed also “Former” Clients at the very first request. A simple Refresh caused it to become “correct” showing only active clients.

This was because an empty ChoiceListField was sometimes an empty string ('') and sometimes None. In get_request_queryset I tested for ar.param_values.client_state is None. The application code was right: a ChoiceListField with blank=True should never have an empty string as value. Solved by setting empty_strings_allowed = False on ChoiceListField.
